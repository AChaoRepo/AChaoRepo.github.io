从零发布 GitHub Pages 个人博客

GitHub Pages 的核心很简单：仓库里有入口文件，Pages 知道从哪里服务它，浏览器就能访问到。

先确认发布模型

个人主页站点通常使用 username.github.io 这个仓库名，并从仓库根目录读取 index.html。如果仓库属于同名账号，访问路径就是 https://username.github.io。

如果仓库名和账号名不一致，它更像一个项目站点，默认访问路径会带上仓库名。上线前先把这件事确认清楚，能省掉很多迷惑时间。

保持静态文件可直接运行

这个博客没有引入构建流程，所有页面都用相对路径引用资源。这样做的好处是本地双击能预览，推送到 GitHub Pages 后也不需要额外安装依赖。

index.html
assets/styles.css
assets/script.js
posts/github-pages-start.html

上线前检查

我会检查三件事：根目录是否有 index.html，文章页资源路径是否正确，移动端导航和卡片是否还能稳定排版。全部通过后再提交推送。
